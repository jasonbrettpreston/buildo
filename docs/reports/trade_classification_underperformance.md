# Trade Classification Underperformance Analysis

## 1. Executive Summary

This report addresses the concern regarding the overall trade classification rate stalling at **44.1%** for the total 133,987 active permits. We performed a database metric analysis and codebase review to answer three specific operational questions:

1. What is the classification rate for residential permits?
2. Are tags currently being used in the engine?
3. Why is "Tier 3" failing to classify any trades?

---

## 2. Investigation Findings

### Question 1: What is the trade classification rate for residential permits?

**Metric Analysis:**
The 44.1% overall rate is misleading when looking at residential versus commercial/generic permits. The database reveals the following breakdown:

* **Total Residential Permits:** 60,988
* **Classified Residential Permits:** 55,280
* **Residential Classification Rate:** **90.6%**

**Why is the overall rate only 44.1%?**
The classification engine heavily favors residential work and specific major additions. The remaining ~73,000 unclassified permits are dragging the average down. These consist largely of:
1. **Commercial / Industrial / Institutional permits** (which do not trigger our residential tags).
2. **Minor mechanical/generic permits** (e.g., HVAC replacements, generic plumbing repairs, minor interior alterations in high-rises) that lack descriptive detail.
3. **Party Wall / Administrative Permits** which do not equate to actual physical trade work.

### Question 2: Are we using the tags?

**Yes.** A review of the `src/lib/classification/classifier.ts` codebase reveals that the tags generated by `classifyScope()` are actively being fed into the classification pipeline. The function `matchTagMatrix()` uses these scope tags to assign trades.

### Question 3: Why isn't Tier 3 classifying any trades?

**Codebase Analysis:**
Tier 3 is reporting zero matches because **Tier 3 has been deprecated and removed from the active classification pipeline.** 

Previously, the classification system was:
* **Tier 1:** Permit Type Exact Match
* **Tier 2:** `work` field Regex matching
* **Tier 3:** `description` field Regex matching

When the system was upgraded to generate robust architectural tags (e.g., `new:kitchen`, `new:bathroom`), it was determined that running messy Regex over the raw description text (Tier 3) was redundant and resulted in false positives. The codebase was refactored so that the `matchTagMatrix()` function completely replaced Tier 2 and Tier 3. 

Currently, all tag-based matches are hardcoded to output to the database as **Tier 2**. Therefore, Tier 3 technically no longer exists in execution.

---

## 3. Improvement Strategy

To push the **overall classification rate** up from 44.1% without breaking the excellent 90.6% residential rate, we must focus exclusively on the non-residential and generic permit backlog.

### 1. Expand Commercial Tags (Scope Expansion)
Currently, `src/lib/classification/scope.ts` generates brilliant tags for "New Houses" and "Residential Additions", but lacks sophisticated tagging for commercial build-outs. 
* **Action:** We must write new tag extractors for commercial terminologies (`tenant-fitout`, `demising-wall`, `commercial-kitchen`, `restaurant-alteration`).

### 2. Broaden the Tag-to-Trade Matrix (Matrix Expansion)
As noted in prior reports, our 20 base trades are too structurally focused.
* **Action:** Implement the missing trades (Millwork/Cabinetry, Tile & Stone, Commercial HVAC, Commercial Glass/Glazing) into the mapping matrix so that tags like `tenant-fitout` have a trade to attach to.

### 3. Graceful Fallbacks for Generic Permits (Tier 1 Tune-Up)
Thousands of permits are simply coded as "Minor Alterations" or "Plumbing" with bare descriptions like "fix pipes".
* **Action:** Ensure Tier 1 rules aggressively assign foundational trades (e.g., Plumbing) to minor permits even if the description lacks enough detail to generate a tag.
